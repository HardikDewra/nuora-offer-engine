# A/B Testing Methodology

## Testing Platform
- **Tool:** Elevate A/B (elevateab.app)
- **Split:** 50/50
- **Device:** Mobile only
- **Country:** United States only
- **Visitors:** All (new + returning)

## How Tests Are Run
1. Control always runs on the main PDP: `mynuora.com/products/feminine-balance-gummies-1`
2. Variant runs on a separate URL (e.g., `/products/nuora-vaginal-probiotic-super-deal`)
3. Traffic is split 50/50 at the page level
4. Tests run for 12-24 hours minimum

## Metrics Tracked

### Primary
- Conversion Rate (%)
- Revenue ($)
- Revenue Per Visitor ($)
- Orders
- AOV ($)

### Secondary
- Add to Cart Rate (%)
- Reached Checkout Rate (%)
- Subscription vs One-Time Purchase split
- Average Session Duration (sec)

## Statistical Requirements
- Minimum ~1,000 visitors per variation
- Probability to win tracked in real-time
- Tests continue until statistical significance is reached

## Key Principles

### One Variable at a Time
All 7 variants tested so far changed 5+ elements simultaneously. Future tests should change exactly ONE element to isolate causation.

### Always Run Concurrent Control
The control's conversion rate varies (5.94% - 8.99%) due to traffic quality fluctuations. Never compare against historical numbers.

### Track Subscription Rate as Primary Metric
For a subscription-first business, subscription rate is as important as overall conversion.

### Minimum Test Duration
At least 2,000 visitors per variation. At current traffic levels, this means 12-15 hours.
